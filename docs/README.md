# Curated knowledge vault

Layer-2 memory for the hub: **what the workspace systems are, how they work, and why**. Authoritative for current project knowledge (see the source-of-truth precedence in `CLAUDE.md`).

## Git policy

`docs/` is tracked **except** `docs/generated/**` and `docs/memory/**` (reproducible / machine-local bookkeeping, git-ignored aside from their READMEs). Curated knowledge is shared and reviewable in git.

## Folders

- `docs/graph/` — Nodes and relationships of the workspace: repos, services, packages, databases, external systems.
- `docs/architecture/` — Current architectural descriptions and durable design rationale.
- `docs/architecture/decisions/` — Durable decisions with context, alternatives and consequences.
- `docs/domain/` — Terminology, entities, business rules and invariants not inferable from structure alone.
- `docs/workflows/` — Normal end-to-end system workflows: trigger, stages, artefacts, completion.
- `docs/runbooks/` — Operational procedures for abnormal situations: diagnosis and recovery.
- `docs/interfaces/` — Contracts between components: APIs, events, queues, schemas, compatibility.
- `docs/operations/` — Knowledge required to run systems in production.
- `docs/development/` — Contributor-facing engineering guidance and hard-to-discover commands.
- `docs/generated/` — Mechanically derived docs. DO NOT hand-edit — regenerated by workflows.
- `docs/memory/` — Bookkeeping that keeps curated docs in sync with source. Not project documentation.
